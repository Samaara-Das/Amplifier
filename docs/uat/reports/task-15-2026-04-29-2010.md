## Bugs found by AC15 that pytest-only UAT missed

### Bug 1 — Wizard "Activate Campaign" bypassed the quality gate entirely (CRITICAL)

The `/company/campaigns/new` POST handler ran:
```python
campaign = Campaign(..., status=campaign_status, ...)
db.add(campaign)
if campaign_status == "active":
    company.balance = float(company.balance) - budget
await db.flush()
```

No quality gate check. A user clicking "Activate Campaign" in the wizard could activate any campaign — including one with `assets_provided=0` (which the activate-API enforces hard-fail on). Real test: wizard activated campaign 58 with assets_provided=0 even though `score_campaign()` returned `passed=False`.

**Fix (commit 9863100)**: gate runs before debiting balance. On `passed=False`, roll back the create and re-render the wizard with feedback.

### Bug 2 — CSRF middleware crashed on multipart file uploads (CRITICAL)

`server/app/core/csrf.py:71` decoded every "form" content-type body as utf-8 and ran `parse_qs()`. `multipart/form-data` requests with file uploads contain binary content — `body.decode()` raised `UnicodeDecodeError`, returning HTTP 500 from the middleware before the route ever ran.

```
File "/home/amplifier/app/server/app/core/csrf.py", line 71, in __call__
    form_data = parse_qs(body.decode())
UnicodeDecodeError: 'utf-8' codec can't decode byte 0xff in position 140: invalid start byte
```

**Fix (commit a5862e5)**: tightened the middleware match from `"form" in ct` → `"application/x-www-form-urlencoded" in ct`. Multipart upload routes (e.g. `/company/campaigns/upload-asset`) handle CSRF inside the route handler now. Verified post-fix: same endpoint goes 500 → 401 (middleware passes through).

### Bug 3 — UX gaps when gate rejects (MEDIUM, follow-up)

When the wizard's "Activate Campaign" gets rejected by the gate:
- Form drops back to step 1 with all fields empty (user loses 4 steps of input)
- Error message from `_render(error=...)` not surfaced anywhere visible in the page (template doesn't show `{{ error }}` at top)
- No "Quality Score" widget or feedback bullets shown — user has no idea WHY rejection happened

Filed as follow-up. Functional behavior is correct (gate fires, rollback works); UX polish needed.

## Follow-ups (filed as separate tasks)

- **Wizard error UX**: when gate rejects via `_render(error=..., status_code=422)`, the template should display the error string at the top of the form AND preserve the user's field values. Currently the form re-renders empty.
- **Wizard image upload UID gap**: the `<input type="file" id="imageFileInput">` is hidden (display:none / pointer-events:none) so Chrome DevTools MCP `take_snapshot` doesn't return its UID. Future UAT runs should bypass via `evaluate_script` to set the file programmatically OR the wizard's upload zone should expose a discoverable a11y label on the visible click-target.
- **AI wizard fallback latency**: when Gemini is 503-throttled, the wizard takes 90-180s to fall through to Mistral. UI shows just "Generating your campaign..." with no progress indicator. Add a timeout warning at 60s + a "Cancel and edit manually" link.
