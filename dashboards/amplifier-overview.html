What Amplifier is

Amplifier is a marketplace. Companies pay to have real people post about their products on social media. Users (we call them Amplifiers) install our app, get matched to campaigns that fit them, post the content, and earn money based on how the post performs. We take 20% of every payout.

Architecture — three components

User App

Runs on user's PC

Lives on the user's Windows computer. Logs into their social media accounts in a real browser, generates the post content using AI, posts it, and tracks how it performs.

Owns: browser logins, AI keys, draft queue, posting, scraping.

Amplifier Server

Currently offline

The brain in the cloud. Holds all campaigns, matches users to campaigns, calculates earnings, holds the money ledger, and serves the web dashboards for companies and admins.

Owns: campaigns, matching, billing, payouts, ledgers.

Companies

Web only

Log into our website. Create campaigns, top up their balance via Stripe, see how their posts are performing, and pay only for real engagement.

Touchpoints: company dashboard, AI campaign wizard, billing.

Data ownership — the trust promise

Stays on the user's PC (never sent anywhere)

  • Browser logins for X / LinkedIn / Facebook / Reddit (cookies, 2FA sessions)
  • Their personal AI API keys (encrypted on disk)
  • Draft posts, scheduled posts, generated images

On the server

  • Campaigns, invitations, assignments
  • Earnings ledger, payouts, trust score, reputation tier
  • Post records (URL, platform, when posted) and scraped engagement metrics

External services we depend on

Supabase — database (Postgres) + file storage Stripe — money in (Checkout) + money out (Connect) Gemini / Mistral / Groq — AI for writing posts (with fallback chain) Cloudflare / Together / Pollinations — AI for images (with fallback) Playwright — the tool that drives the browser to post Vercel — old hosting, abandoned

How the product actually works — step by step

Four walkthroughs: how a company uses Amplifier, how a new user gets set up, what a user does day-to-day, and what the agent does silently on the user's computer.

A · A company uses Amplifier

From signup to seeing real posts in the wild

  1. 1. Sign up. Marketer goes to /company/login, registers with email + password, gets a session token. No paperwork, no sales call.
  2. 2. Top up balance. They click "Add Funds" on the Billing page, pay via Stripe Checkout (minimum $50). Stripe webhook credits their balance. They can't run campaigns until they have money on hand.
  3. 3. Create a campaign — two options.
    • AI wizard (the easy way): paste 1–3 of your website URLs. Server scrapes them, Gemini reads everything, and writes a draft campaign for you — title, brief, content guidance, suggested payout rates, suggested target niches. The marketer reviews, edits if needed, saves.
    • Manual form: fill in everything yourself — product info, brief, target platforms, follower minimums, target regions, budget, payout rates per metric (cents per like, cents per 1000 views, cents per comment, cents per repost).
  4. 4. Quality gate (Task #15 — not built yet). When they click "Activate", the server will score the campaign 0–100. Below 85 = blocked with specific feedback ("brief is too short", "no payout rates set"). Above 85 = AI also checks for harmful content or product/audience mismatch. Until #15 ships, weak campaigns can go live without checks.
  5. 5. Server matches users. Once active, the server runs the matching engine: hard filters (right platform, enough followers, right region, not over their tier limit) then Gemini scores each candidate 0–100 on relevance. Anyone above 40 gets an invitation; the rest are skipped.
  6. 6. What the company sees on their dashboard (10 pages). Login · Overview · Campaigns list · Create campaign · AI wizard · Campaign detail (live stats, creators reached, posts made, impressions, engagement, budget used/remaining, invitation funnel, decline reasons) · Billing · Influencers (top creators on their campaigns) · Stats · Settings.
  7. 7. Pay only for results. They never pay per impression upfront. They pay only when scraped metrics come in — likes, views, comments, reposts. If posts under-perform, less budget is consumed. Unused budget refunds when the campaign ends or is cancelled.

B · A user (Amplifier) sets up the app

First-run experience — installer to first campaign in ~10 minutes

  1. 1. Download & install. They visit our website, click "Download Amplifier", get AmplifierSetup.exe (~350MB — bundled Chromium browser inside). Double-click, Inno Setup wizard, install to C:\Program Files\Amplifier\, finish. (Installer not built yet — Task #20.)
  2. 2. First launch. A small icon appears in the Windows system tray. The agent opens their default browser to our register/login page (or, if they signed up earlier, prompts them to log in).
  3. 3. Log in. They register with email + password. The web page hands a token back to the local agent (via a localhost callback URL). The agent encrypts and stores the token. They never have to log in again on that machine.
  4. 4. Connect each social platform. They right-click the tray icon → "Connect Platforms". For each platform (LinkedIn, Facebook, Reddit), the app opens a real browser window pointed at that platform's login page. The user types their password and 2FA themselves. We never see it. When they're logged in, we save the browser session (cookies) into a folder on their machine. From now on, posting is automatic — they don't log in again unless the session expires.
  5. 5. Profile scan (automatic). The agent quietly visits their profile on each connected platform and reads: follower count, bio, recent posts, engagement rates. AI then figures out what topics they post about. We try cheap text reading first; only use the expensive image-AI if text fails.
  6. 6. Pick niches, region, mode. The user confirms or adjusts the niches AI detected, picks their region, and chooses a mode:
    • Semi-auto — agent generates drafts, user reviews and approves before posting.
    • Full-auto — agent generates and posts without asking. For users who trust the AI.
  7. 7. Enter AI API keys. They paste in a Gemini key (free tier works), optionally Mistral and Groq for fallback. The agent tests each key with a tiny request — green check or red X. Keys are encrypted on disk with a key derived from their machine. They never leave the computer.
  8. 8. Done. Background agent starts. From this moment, it polls our server for campaigns, generates content, posts at the right times, and scrapes metrics — all without the user opening anything.

C · A user's daily life on Amplifier

What actually happens day-to-day, from waking up to getting paid

  1. 1. Morning — invitations arrive. Every 10 minutes the agent quietly polls the server for new campaign matches. When new ones come in, the user gets a small desktop notification: "You have 2 new campaign invitations."
  2. 2. They open the app and review invites. Each invitation card shows the campaign brief, the company, the payout rates ("$0.01 per like, $0.50 per 1k views"), required platforms, and a live countdown until the invite expires. They click Accept or Reject (with a quick decline reason).
  3. 3. Drafts are generated automatically. The agent runs the 4-phase content engine in the background. For each accepted campaign, it produces one draft per platform per day — different angle, different hook, different length, sometimes with an AI-generated image based on the company's product photo.
  4. 4. They review drafts (semi-auto users only). In the dashboard, today's drafts appear card-by-card: LinkedIn version, Facebook version, Reddit title + body. They can edit the text, swap the image, reject it (agent regenerates), or approve. Full-auto users skip this step entirely.
  5. 5. Posts go out at peak hours (US-aligned). The agent posts at 6 fixed slots optimised for US audiences:
    Slot 1 — 8:00 AM ESTSlot 2 — 10:00 AM ESTSlot 3 — 1:00 PM EST Slot 4 — 3:00 PM ESTSlot 5 — 6:00 PM ESTSlot 6 — 8:00 PM EST
    When a slot arrives, the agent opens the user's logged-in browser, types the post like a human (slow typing, scrolling, mouse movement), uploads any image, and clicks Post.
  6. 6. Agent captures the post URL. Right after posting, the agent finds the link to the new post and saves it. We need that URL to revisit later and read the engagement.
  7. 7. Metrics start arriving. Every 24 hours after posting, the agent revisits each post URL and reads likes, comments, views, reposts. The numbers are sent to the server.
  8. 8. Earnings tick up. The server multiplies each metric by the campaign's payout rate, takes our 20% cut, and credits the user. New earnings show as Pending for 7 days — the hold protects against fake metrics or deleted posts.
  9. 9. Withdraw. After the 7-day hold, earnings move to Available. Once available balance hits $10+, they click Withdraw, money is sent to their bank via Stripe Connect, typically arriving in 2–3 business days. (Stripe payouts not live yet — Task #19.)
  10. 10. Tier up over time. After 20 successful posts they're promoted to Grower (more campaigns at once). After 100 posts + good trust score they hit Amplifier tier — earnings double.

How each task works — by batch

Each task explained in 2-3 plain sentences. Status: ✅ done · ⏳ pending · ⏸ deferred.

Batch 1 — Money Loop

4 / 4 done

#1 Capture post URL ✅ — After the user posts, the agent grabs the link to the new post (e.g. the LinkedIn URL). Without this we can't find the post later to read its likes and comments. Each platform stores the URL differently, so we have a different trick per platform.

#9 Metric scraping ✅ — Every 24 hours, the agent revisits each posted URL and reads the engagement numbers (likes, views, comments). Reddit and X show view counts; LinkedIn and Facebook hide them. We also detect "this post was deleted" so we don't bill for vanished posts.

#10 Billing ✅ — When metrics arrive, the server multiplies them by the payout rates (cents per like, cents per 1000 views, etc.), takes our 20% cut, and credits the user. Earnings stay locked for 7 days before withdrawal — gives us time to void if the post turns out fake or gets deleted.

#11 Earnings display ✅ — The user's app shows total earned, available balance, and pending balance (still in the 7-day hold). When available balance hits $10+, a Withdraw button lets them cash out.

Batch 2 — AI Brain

3 / 4 done

#13 Profile scraping ✅ — When the user connects a platform, our agent reads their profile (followers, bio, recent posts, engagement) and uses AI to figure out what topics they post about. This data feeds into matching. We try to do it cheaply: read the page text first, only use the expensive image-AI as a last resort.

#12 AI matching ✅ — When a campaign goes live, the server checks each user against it. First it filters with hard rules (right platform, enough followers, right region). Then Gemini scores fit 0–100 based on whether the user's content is actually relevant. Anyone scoring below 40 is skipped — no point inviting them.

#14 Content agent ✅ — Generates posts in 4 phases: (1) Research — read the company's website and find recent niche news; (2) Strategy — pick the angle for today based on the campaign goal (awareness, leads, virality, engagement); (3) Creation — write a different version per platform; (4) Review — user approves, or auto-post in full-auto mode.

#15 Quality gate ⏳ NOT BUILT — Will check campaigns before they go live. Two layers: an instant rule check (is the brief long enough? is the budget at least $100? are payout rates set?) plus an AI review (is the brief coherent? does the targeting match the product? anything harmful?). Campaigns scoring below 85/100 will be blocked with feedback telling the company exactly what to fix.
